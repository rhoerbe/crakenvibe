# Governance: single maintainer, RFC/ADR process, DCO

At zero external contributors, heavyweight governance would be process cosplay; at the first external contributor, no process is a trap. Decision: single maintainer for now; spec changes go through RFC-labeled PRs and ADRs; the spec itself is semver'd and tagged v1.0-rc only after all four v1 connectors have validated it (implementation-validated, never committee-first). Contributions are accepted under DCO — no CLA. A steering group is deferred until multi-vendor interest is real; foundation approaches (OpenSSF/LF) wait for traction.
